# JAKIM Hijri ICS Calendar Generator

A fully automated, open-source project that fetches the official Takwim Hijri data from Malaysia's **e-Solat (JAKIM)** and generates a downloadable `.ics` file. This allows you to subscribe to an accurate, JAKIM-compliant Hijri calendar directly in Google Calendar, Outlook, or Apple Calendar.

---

## 👥 For Public Users (How to Subscribe)
You don't need to touch any code! You can simply subscribe to the calendar generated by this project.

1.  Open **Google Calendar** on a computer.
2.  On the left panel, click the **"+"** button next to **"Other calendars"**.
3.  Select **"From URL"**.
4.  Paste the following link:
    `https://raw.githubusercontent.com/multimedia-mamtj6/dev/main/calendar/hijri/ics/jakim_hijri.ics`
5.  Click **Add calendar**. 

*The Hijri dates will now appear as all-day events on your calendar automatically.*

## 🛠 For Developers (How to Customize)
This project is built using Python. You can modify `calendar/hijri/ics/generate_ics.py` to change how the data is displayed.

### Prerequisites
*   Python 3.x installed
*   `pip install requests`

### Running Locally
1. Clone the repository: `git clone https://github.com/multimedia-mamtj6/dev.git`
2. Navigate to the script folder: `cd calendar/hijri/ics/`
3. Run the generator: `python generate_ics.py`
4. The file `jakim_hijri.ics` will be generated in the same folder.

### Key Customization Points
*   **Change Zone:** Edit the `url` in `fetch_waktusolat_data()` to change the location (e.g., from `WLY01` to `JHR01`).
*   **Format Titles:** Modify the `summary` variable in `generate_calendar()` to include prayer times or additional text.
*   **Links:** Update the `DESCRIPTION` variable to include your own links or custom messages.

## ⚙️ Maintenance & Automation
This repository uses **GitHub Actions** to keep the calendar updated without human intervention.

### How it works:
*   **Automation:** The script runs automatically via `.github/workflows/update_calendar.yml` on the **1st of every month**.
*   **Data Source:** It fetches data from `api.waktusolat.app`, a reliable mirror of JAKIM’s data that avoids international IP geo-blocking.
*   **Storage:** The generated `.ics` file is stored in `calendar/hijri/ics/jakim_hijri.ics`.

### Troubleshooting
*   **Bot Permissions:** If the GitHub Action fails with `403 Permission Denied`, go to: 
    *   *Settings > Actions > General > Workflow permissions*
    *   Set to **"Read and write permissions"** and click Save.
*   **API Down?** If the mirror API is down, the Action will fail. You can trigger a re-run manually by going to the **Actions** tab, selecting the workflow, and clicking **"Run workflow"**.
